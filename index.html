Button accessible name tests

The purpose these tests is to show the cascade order of how accessible names are applied in the accessibility tree for the button element. These examples should not be considered proper practice.

  • → button where "aria-labelledby" should be accessible name
  • → button where "aria-label" should be accessible name
  • → button where content should be accessible name
  • → button where "title" should be accessible name
  • → button with no accessible name

ARIA banner tests - Added 10/07/2021

When the header element is scoped to the body element (not a descendant of article, aside, main, nav, section or any element with a role of article, complementary, main, navigation, region) it represents introductory content for the page as a whole. In these cases, the header element will have an implicit ARIA role of banner.

These test cases are designed to determine how the header element and elements with a role of banner are announced by various screen readers, in specific situations.

  • → header
  • → role="banner
