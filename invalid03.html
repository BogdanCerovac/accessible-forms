input with aria-invalid="grammar" attribute

Return to main list

This is a test case designed to determine how the aria-invalid attribute, applied to the input element, is announced by various screen readers.

Example

Code

<div>
  <label for="name">Name</label>
  <input type="text" id="name" aria-invalid="grammar">
</div>

Assistive technologies

VoiceOver

  • Chrome: Name. Invalid grammar. Edit text.
  • Firefox: Name. Invalid grammar. Edit text.
  • Safari: Name. Invalid grammar. Edit text.

NVDA

  • Chrome: Name. Edit. Invalid entry. Grammar. Error. Blank.
  • Firefox: Name. Edit. Invalid entry. Has autocomplete. Grammar. Error. Blank.
  • Edge: Name. Edit. Invalid entry. Grammar. Error. Blank.

JAWS

  • Chrome: Name. Edit. Invalid entry. Type in text. [Grammar not announced]
  • Firefox: Name. Edit. Invalid entry. Type in text. [Grammar not announced]
  • Edge: Name. Edit. Invalid entry. Type in text. [Grammar not announced]
